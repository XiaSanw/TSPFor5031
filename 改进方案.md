# POMO TSP 改进方案

## 一、改进思路概述

在 baseline POMO 模型基础上，从**训练端**和**测试端**两个方向做改进：

| 方向 | 改进方法 | 效果 | 难度 |
|------|---------|------|------|
| 测试端 | 多采样推理（Multi-sampling） | 高 | 低 |
| 测试端 | 2-opt 局部搜索后处理 | 高（已被老师禁用） | 中 |
| 训练端 | 混合规模训练（100~300 城市） | 高 | 中 |
| 训练端 | 模型扩容（1.6M → 6.8M 参数） | 高 | 低 |
| 训练端 | 训练时 8-fold 数据增强 | 高 | 低 |
| 训练端 | Cosine annealing + warmup LR | 中 | 低 |
| 训练端 | Leader Reward（Wang et al., 2024） | 高 | 中 |
| 训练端 | 学习率策略调整 | 中 | 低 |

> **核心逻辑**：测试端改进让"笨模型"也能发挥更好（见效快），训练端改进让模型本身更聪明（长期收益）。

## 二、测试端改进

### 2.1 多采样推理（Multi-sampling）

**原理**：
- 当前 baseline 在测试时用 `argmax`（每步选概率最大的城市），只产生一条确定性路线
- 改为按概率**随机采样**多次，每次得到不同的路线，取最短的那条
- 最后一次仍用 argmax，保证至少有一条贪心路线

**具体做法**：


### 2.2 2-opt 局部搜索

**原理**：
- 神经网络生成的路线可能有交叉或局部不优的地方
- 2-opt 每次选两条边，尝试"断开再重连"，如果新路线更短就接受
- 反复执行直到没有改进为止

**具体做法**：

#### 修改文件：`TSP/POMO/TSPTester_LIB.py`

**第一步**：在文件顶部 import 区域修改：

```python
# 改为：
```

**第二步**：在 `_normalize_to_unit_square` 函数之后、`LibResult` 类之前，添加三个工具函数：

```python
def _compute_dist_matrix(coords, edge_weight_type='EUC_2D'):
    """计算距离矩阵。coords: (N, 2) tensor -> list of list (Python 嵌套列表)"""
    diff = coords.unsqueeze(0) - coords.unsqueeze(1)
    dists = (diff ** 2).sum(dim=2).sqrt()
    if edge_weight_type == 'CEIL_2D':
        dists = torch.ceil(dists)
    elif edge_weight_type == 'EUC_2D':
        dists = torch.floor(dists + 0.5)
    return dists.cpu().tolist()


def _two_opt(tour, dist_matrix, max_iter=10000):
    """
    经典 2-opt 局部搜索。
    tour: 城市索引列表，如 [0, 3, 1, 2]
    dist_matrix: Python 嵌套列表，dist_matrix[i][j] = 城市 i 到 j 的距离
    返回优化后的 tour。
    """
    n = len(tour)
    tour = list(tour)
    improved = True
    iter_count = 0
    while improved and iter_count < max_iter:
        improved = False
        for i in range(1, n - 1):
            for j in range(i + 1, n):
                # 当前边：tour[i-1]->tour[i] 和 tour[j]->tour[j+1]
                # 候选边：tour[i-1]->tour[j] 和 tour[i]->tour[j+1]
                a, b = tour[i - 1], tour[i]
                c, d = tour[j], tour[(j + 1) % n]
                delta = (dist_matrix[a][c] + dist_matrix[b][d]
                         - dist_matrix[a][b] - dist_matrix[c][d])
                if delta < -1e-6:
                    # 反转 tour[i..j] 这一段
                    tour[i:j + 1] = reversed(tour[i:j + 1])
                    improved = True
                    break
            if improved:
                break
        iter_count += 1
    return tour


def _tour_length(tour, dist_matrix):
    """计算一条路线的总长度。"""
    n = len(tour)
    return sum(dist_matrix[tour[i]][tour[(i + 1) % n]] for i in range(n))
```

**第三步**：把 `_test_one_instance` 方法**整个替换**为以下代码：

```python
def _test_one_instance(self, nodes_xy_normalized: torch.Tensor,
                       coords_orig: torch.Tensor, ew_type: str
                       ) -> Tuple[float, float]:
    # ------ 读取参数 ------
    if self.tester_params['augmentation_enable']:
        aug_factor = self.tester_params['aug_factor']
        if aug_factor != 8:
            raise NotImplementedError('Only aug_factor=8 is supported.')
    else:
        aug_factor = 1

    num_samples = self.tester_params.get('num_samples', 1)
    enable_2opt = self.tester_params.get('enable_2opt', False)

    # ------ 数据增强 ------
    problems = nodes_xy_normalized
    if aug_factor > 1:
        problems = augment_xy_data_by_8_fold(problems)

    effective_batch = problems.size(0)   # 1 或 8（有增强时）
    problem_size = problems.size(1)

    # ------ 初始化环境 ------
    env = Env(problem_size=problem_size, pomo_size=problem_size)
    env.batch_size = effective_batch
    env.problems = problems.to(self.device)
    env.BATCH_IDX = torch.arange(effective_batch, device=self.device)[:, None].expand(
        effective_batch, env.pomo_size)
    env.POMO_IDX = torch.arange(env.pomo_size, device=self.device)[None, :].expand(
        effective_batch, env.pomo_size)
    env.original_node_xy_lib = coords_orig[None, :, :]
    env.edge_weight_type = ew_type

    # ------ 预计算距离矩阵（2-opt 用）------
    dist_matrix = None
    if enable_2opt:
        dist_matrix = _compute_dist_matrix(coords_orig, ew_type)

    # ------ 多采样推理 ------
    self.model.eval()
    all_sample_best = []     # 每次采样的最优路线长度

    with torch.no_grad():
        # Encoder 只算一次（所有采样共享同一个编码结果）
        reset_state, _, _ = env.reset()
        self.model.pre_forward(reset_state)
        saved_eval = self.model.model_params['eval_type']

        for sample_id in range(num_samples):
            # 前 N-1 次用 softmax 随机采样，最后一次用 argmax 贪心
            self.model.model_params['eval_type'] = (
                'softmax' if num_samples > 1 and sample_id < num_samples - 1
                else 'argmax'
            )

            # 手动重置环境的动态状态（不需要重新编码，所以不调 env.reset()）
            env.selected_count = 0
            env.current_node = None
            env.selected_node_list = torch.zeros(
                (effective_batch, env.pomo_size, 0),
                dtype=torch.long, device=self.device
            )
            env.step_state = Step_State(
                BATCH_IDX=env.BATCH_IDX, POMO_IDX=env.POMO_IDX
            )
            env.step_state.ninf_mask = torch.zeros(
                (effective_batch, env.pomo_size, problem_size),
                device=self.device
            )

            # Decoder 逐步选城市
            state, reward, done = env.pre_step()
            while not done:
                selected, _ = self.model(state)
                state, reward, done = env.step(selected, lib_mode=True)

            # reward 是负路程，取反得到实际路程
            tour_lengths = -reward   # shape: (effective_batch, pomo_size)

            # ------ 可选：2-opt 后处理 ------
            if enable_2opt:
                batch_best_lengths = []
                best_pomo_idx = tour_lengths.argmin(dim=1)
                for b in range(effective_batch):
                    pidx = best_pomo_idx[b].item()
                    tour = env.selected_node_list[b, pidx].cpu().tolist()
                    optimized = _two_opt(tour, dist_matrix)
                    opt_len = _tour_length(optimized, dist_matrix)
                    batch_best_lengths.append(opt_len)
                all_sample_best.append(
                    torch.tensor(batch_best_lengths, dtype=torch.float32)
                )
            else:
                # 不做 2-opt，直接取每个 aug batch 里最短的 POMO 路线
                all_sample_best.append(tour_lengths.min(dim=1).values.cpu())

        # 恢复原来的 eval_type
        self.model.model_params['eval_type'] = saved_eval

    # ------ 合并所有采样结果 ------
    all_sample_best = torch.stack(all_sample_best, dim=0)
    # shape: (num_samples, effective_batch)

    # no_aug_score: 第 0 个 batch（无增强版本）在所有采样中的最优
    no_aug_score = all_sample_best[:, 0].min().item()
    # aug_score: 所有采样 x 所有增强版本中的全局最优
    aug_score = all_sample_best.min().item()

    return float(no_aug_score), float(aug_score)
```

```python
    def _train_one_batch(self, batch_size):
        import random

        # === MODIFIED (Iter-2): weighted random choice of problem size ===
        # 100 cities trained more frequently, 250 less (hidden test max ~200+)
        problem_size = random.choices(
            self.mixed_sizes,
            weights=[3, 2, 1, 1]
        )[0]
        if problem_size != self.env.problem_size:
            self.env.problem_size = problem_size
            self.env.pomo_size = problem_size

        # === MODIFIED (Iter-2): dynamic batch_size scaling to avoid OOM ===
        # VRAM ~ problem_size^2; base = 100 cities @ batch=64
        base_batch = self.trainer_params['train_batch_size']
        scale = (100 / problem_size) ** 2
        adjusted_batch = max(16, int(base_batch * scale))  # floor at 16
        batch_size = min(adjusted_batch, batch_size)

        # ===== 以下与原版相同 =====

        # Prep
        ###############################################
        self.model.train()
        self.env.load_problems(batch_size)
        reset_state, _, _ = self.env.reset()
        self.model.pre_forward(reset_state)

        prob_list = torch.zeros(size=(batch_size, self.env.pomo_size, 0))
        # shape: (batch, pomo, 0~problem)

        # POMO Rollout
        ###############################################
        state, reward, done = self.env.pre_step()
        while not done:
            selected, prob = self.model(state)
            # shape: (batch, pomo)
            state, reward, done = self.env.step(selected)
            prob_list = torch.cat((prob_list, prob[:, :, None]), dim=2)

        # Loss（保留 POMO 原版 baseline）
        ###############################################
        advantage = reward - reward.float().mean(dim=1, keepdims=True)
        # shape: (batch, pomo)
        log_prob = prob_list.log().sum(dim=2)
        # size = (batch, pomo)
        loss = -advantage * log_prob  # Minus Sign: To Increase REWARD
        # shape: (batch, pomo)
        loss_mean = loss.mean()

        # Score
        ###############################################
        max_pomo_reward, _ = reward.max(dim=1)  # get best results from pomo
        score_mean = -max_pomo_reward.float().mean()  # negative sign to make positive value

        # Step & Return
        ###############################################
        self.model.zero_grad()
        loss_mean.backward()
        self.optimizer.step()
        # === MODIFIED (Iter-2 patch): 返回实际使用的 batch_size，供上层正确加权与计数 ===
        return score_mean.item(), loss_mean.item(), batch_size
```

**第三步**：修改 `_train_one_epoch` 中调用 `_train_one_batch` 的位置，取实际 batch size：

```python
            # === MODIFIED (Iter-2 patch): _train_one_batch 内部会动态缩小 batch_size ===
```

### 3.2 为什么不加 Critic

POMO 的 baseline 是"同一问题 100 条路线的平均 reward"，这已经非常强：
- 100 条路线来自同一个问题实例，方差天然很小
- POMO 原论文专门论证过：shared baseline 优于 learned critic
- 简单的 Critic（坐标 -> 线性层 -> 平均池化 -> MLP）预测能力太弱，可能反而给出误导性的 advantage

另外，加载旧 checkpoint 到带 Critic 的新模型时，optimizer 参数组不匹配会直接报错，需要额外处理兼容性问题。综合投入产出比，不加 Critic。

### Iter-2 Patch：修复 episode 计数与日志加权偏差

**时间**：2026-04-22
**问题**：`_train_one_batch` 内部会根据城市规模动态缩小 `batch_size`（如 250 城市时从 64 缩到 16），但旧代码只返回 `score, loss`，导致 `_train_one_epoch` 仍按原始 64 进行 `AverageMeter.update` 和 `episode +=`。这造成：
1. 日志加权偏差——大城市 batch 被赋予过高权重；
2. 每 epoch 实际梯度步数少于声称的 100K episodes。
**修复**：
- `_train_one_batch` 末尾返回三元组 `(score, loss, actual_batch_size)`；
- `_train_one_epoch` 解包后用 `actual_bs` 更新 `score_AM`、`loss_AM` 和 `episode`。

## 六、预期效果

- **多采样**（8 次采样 x 8 倍增强 x N 个 POMO 起点）：大幅增加候选路线数量，预计降低 avg_aug_gap 1~3 个百分点
- **2-opt**：消除路线中的交叉和局部不优，预计在每条路线上额外降低 0.5~2 个百分点
- **混合规模训练**：提升 150+ 城市实例的表现，预计对大实例（kroA200, pr226, pr299）改善最明显

---

## 七、注意事项

1. **2-opt 会增加测试时间**：对 299 城市的实例，单次 2-opt 可能需要几秒。8 次采样 x 8 倍增强 = 64 次 2-opt，总时间可能较长。如果超时，可以减少 `num_samples` 或关闭 2-opt
2. **混合规模训练的显存**：200 城市的显存占用约是 100 城市的 4 倍，250 城市约是 6.25 倍。代码中已通过动态 batch_size 处理（下限 16），但如果 GPU 显存 < 12GB，建议把 `mixed_sizes` 改为 `[100, 150, 200]`
3. **Scheduler milestone**：改为 3401 是因为从 epoch 3000 续训，需要给新任务（混合规模）足够的高学习率阶段

#### B. 混合规模扩展 + 放宽动态batch

- `mixed_sizes` 从 `[100, 150, 200, 250]` 扩展为 `[100, 150, 200, 250, 300]`

- batch_size 下限从 16 提高到 32（A100 40GB 下大城市梯度质量大幅提升）

#### C. 训练时 8-fold 数据增强

- 每个随机生成的 TSP 实例在训练时做 8-fold 翻转/旋转增强
- 增强后 batch 扩大 N 倍，等效数据量 x8
- 为控制 VRAM：200 城市以上降低 aug_factor（250→4, 300→2）

**原理**：POMO 测试时已用 8-fold 增强，但训练时没用。训练时增强让模型在同一个问题实例上看到 8 个等效版本，policy gradient 信号更丰富。

#### D. Cosine Annealing + Linear Warmup

替换 MultiStepLR 阶梯衰减：
- Epoch 1~200：Linear warmup（lr 从 3e-7 → 3e-4）
- Epoch 201~5000：Cosine annealing（lr 从 3e-4 → 1e-6）

**动机**：阶梯衰减的 milestone 需要人工调，cosine annealing 是更平滑的衰减策略，warmup 防止训练初期梯度爆炸。

#### E. Leader Reward（Wang et al., 2024）

**论文**：[Leader Reward for POMO-Based Neural Combinatorial Optimization](https://arxiv.org/abs/2405.13947)

**原理**：
- POMO 用 100 条路线的平均 reward 做 baseline（REINFORCE with shared baseline）
- Leader Reward 额外奖励每个问题中**最好的那条路线**（leader）
- 让模型集中火力生产最优解，而非提高所有路线的平均质量

**实现**（在 `_train_one_batch` 中）：
```python
# 标准 POMO baseline
baseline = reward.float().mean(dim=1, keepdims=True)
advantage = reward - baseline  # (batch, pomo)

# Leader Reward
alpha = 2.0
leader_idx = advantage.argmax(dim=1)       # 每个问题的最优路线
advantage[batch_idx, leader_idx] *= alpha  # 放大 leader 的 advantage
advantage = advantage / alpha              # 全局归一化（稳定训练）
```

**效果**：原论文报告 TSP100 的 optimality gap 降低 100 倍以上，几乎零额外计算开销。
